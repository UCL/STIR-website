STIR at the IEEE MIC 2015 Short Course on Image Reconstruction

This page is for participants in the Short Course on Image Reconstruction at the IEEE MIC. It will be updated information and extra files before the course.
Please read but check back later. Please refresh the page when you re-visit.

During the course, we will have practical exercises using STIR. These exercises are an integral part of the course. To be able to participate, students are asked to bring their own laptop if possible.

We will provide a Virtual Machine with everything preinstalled for the course. Please follow the installation instructions below to get this on your laptop before the conference.. Please note that this will require administrator permissions.

Please note that the STIR version that we will be using is more recent that the current 3.0. Therefore, please use the virtual machine.. If you really want to do this all yourself, you will need to get STIR from github, install SWIG, Python, etc and compile STIR with Python support. You would then need to get the data and excample scripts. Don't try this...

If you want to prepare for these exercises, please check some of the STIR documentation on the website, or register for STIR and download the software already (or get them from github). You could then have a look at the files in examples/PET_simulation provided in the download.

Virtual Machine installation instructions

This text is adapted from the vGate instructions.

We are still in the process of getting this ready. If you are not familiar with VirtualBox, We recommend experimenting with vGate now.

  1. Install VirtualBox
    You do not need to install the Oracle extensions to VirtualBox, although it might come in handy for USB support.
    Although other Virtual Machine software might work, we have not tried this and will not be able to help to get this going.
  2. Download the Virtual Disk Image (VDI) (coming soon).
  3. Once Virtual Box is installed and you have the VDI, do the following:
    1. Launch Virtual Box and click on "New" to create a new virtual machine.
    2. Click on "Next", and give a name to your machine.
    3. Choose 'Linux' as Operating System, and 'Ubuntu' as Version. If you are asked to, precise 64 bits system. Click on "Next".
    4. Select the amount of RAM that you want to give to the virtual machine (at least 1Gb if possible, the higher the better). Click on "Next".
    5. Select "Use existing hard disk" and then go in the menu on the right, click on "Hard drive" "Add" and find the VDI file that you have just downloaded. Click on "Next".
    6. Click on "Finish", and you can change settings of your virtual machine and start it.
    7. The only user in the machine is named "stir" with password "virtual".
  4. Adjust your Ubuntu settings:
    1. The keyboard type is set to English-UK. If this is not ok:
      1. Go into the "System" menu, then in "Preferences" and finally in "Keyboard".
      2. Go in the "Layout" tab and choose the appropriate layout corresponding to your keyboard.
    2. Default settings should allow you to access the internet from in the virtual machine. If not, please check the Virtual Box documentation
    3. You probably want to configure a shared directory between the host and the guest machine such that your virtual machine can "see" your "normal" files. Please read the Virtualbox documentation on Folder Sharing.
