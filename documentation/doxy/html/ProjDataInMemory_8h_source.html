36 /* Implementation note (KT)
37 
38  I first used the std::stringstream class (when available).
39  However, this class currently has a problem that you cannot preallocate
40  a buffer size. This means that when we write to the stringstream, it will
41  grow piece by piece. For some implementations (i.e. those that keep the memory
42  contiguous), this might mean multiple reallocations and copying of data.
43  Of course, for 'smart' implementations of stringstream, this wouldn't happen.
44  Still, I've decided to not take the risk, and always use old style strstream instead.
45 
46  It's not clear if strstream will ever disappear from C++, but in any case, it won't happen
47  very soon. Still, if you no longer have strstream, or don't want to use it, you can enable
48  the stringstream implementation by removing the next line.
49 */
50 #define STIR_USE_OLD_STRSTREAM
