25 /* Implementation note (KT)
26 
27  I first used the std::stringstream class (when available).
28  However, this class currently has a problem that you cannot preallocate
29  a buffer size. This means that when we write to the stringstream, it will
30  grow piece by piece. For some implementations (i.e. those that keep the memory
31  contiguous), this might mean multiple reallocations and copying of data.
32  Of course, for 'smart' implementations of stringstream, this wouldn't happen.
33 
34  So, we now use boost::interprocess::bufferstream instead. You could use
35  use old style strstream instead, but that's now very deprecated, so that's not recommended.
